	Docker сеть (сухой остаток)

	1. Хост (hostname)
	   = имя сервиса в docker-compose
	   пример:
	   services:
		 postgres:
		 redis:

	   → host: postgres
	   → host: redis

	2. IP адрес
	   назначается автоматически Docker
	   пример:
	   postgres → 172.18.0.2
	   redis    → 172.18.0.3

	3. Связка имя → IP
	   делает встроенный DNS Docker
	   (сам, без участия пользователя)

	4. Где смотреть IP
	   docker network inspect <network>

	5. Где проверить соответствие
	   внутри контейнера:
	   getent hosts postgres

	6. Имя контейнера
	   airflowinstance-postgres-1
	   НЕ используется для подключения

	7. Главное правило
	   всегда использовать:
	   host = имя сервиса (postgres, redis)
	   НЕ IP и НЕ имя контейнера
